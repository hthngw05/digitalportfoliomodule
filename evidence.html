Artefact 01

MRP Data Analysis and Planning View

Related workplace project: Material Requirements Planning data analysis and dashboard preparation.

Open PDF artefact ↗

Deliverable

A mock source extract, transformation process and horizontal planning view that organises dated material demand and supply records.

My contribution

I reviewed the source structure, clarified MRP element meanings, cleaned and grouped records, and designed the date-based output.

How it will be used

The structure can support planning review and later dashboard development after the source fields and business rules are validated.

Technical Skills and Competencies (TSC)

Technical skill / competency Level demonstrated Evidence Corrective action / next step
Data Analytics Level 2 - Apply Cleaned, classified and interpreted structured MRP records. Validate manufacturing-specific rules with guidance and approved source data.
Data Visualisation Level 2 - Apply Created a clearer date-based planning view. Conduct user testing to confirm that the layout supports actual planning work.
Business Needs Analysis Level 2 - Apply with guidance Clarified field meanings and output expectations instead of assuming definitions. Continue documenting confirmed requirements, open questions and limitations.

Critical Core Skills (CCS): actions and results

Critical Core Skill Task and action taken Result
Sense Making Compared dates, field meanings and MRP element types. Separated confirmed information from items that still required clarification.
Digital Fluency Used spreadsheet and structured-data techniques to prepare the output. Produced a reusable demonstration structure.
Communication Asked for clarification when field definitions were unclear. Reduced the risk of presenting an incorrect interpretation.
Problem Solving Investigated missing fields and incomplete calculation inputs. Recorded the limitations rather than overstating what the output could support.
Artefact 02

Production Traveller OCR Proof of Concept

Related workplace project: MES and production traveller digitalisation.

Open PDF artefact ↗

Deliverable

A field map, OCR workflow, mock structured output and comparison of two operator-identification methods.

My contribution

I studied the traveller structure, mapped the required fields, tested OCR outputs and documented the recommended identification method.

How it will be used

The proof of concept helps assess whether repetitive data entry can be reduced while keeping manual review for uncertain values.

Technical Skills and Competencies (TSC)

Technical skill / competency Level demonstrated Evidence Corrective action / next step
Data Collection and Analysis Level 2 - Apply Mapped traveller values to structured fields and validation rules. Test the mapping with a larger approved sample containing different document conditions.
Software Testing Level 2 - Apply Compared operator-name OCR with numeric stamp-ID OCR and documented the observed results. Define repeatable test cases, confidence thresholds and exception outcomes.
Process Improvement Level 2 - Apply with guidance Proposed a workflow that reduces repetitive entry while retaining review controls. Validate the workflow with operational users before wider implementation.

Critical Core Skills (CCS): actions and results

Critical Core Skill Task and action taken Result
Problem Solving Investigated why operator names from curved stamps were unreliable. Identified the numeric stamp ID as a simpler and more consistent identifier.
Digital Fluency Used UiPath OCR and structured output formats. Created an extraction method that can be tested again.
Sense Making Compared the OCR output with the visible document context. Avoided accepting text that appeared possible but was incorrect.
Communication Documented the limitation and proposed authorised lookup method. Made the recommendation understandable to technical and operational users.
